IDA CFS Signature Plugin

An IDA Pro plugin that exports and imports function signatures using a custom .cfs format.
It helps reuse function names across similar binaries, game builds, or application versions.

FILES PROVIDED

IDA-Plugin.py
Main IDA plugin script written in Python.
Handles the UI, signature import/export, and settings.

*.cfs
Function signature database files generated by the plugin.
Used to rename functions in other IDA databases.

README
Documentation and usage instructions.

INSTALLATION

Copy the file:
IDA-Plugin.py

Place it in the IDA plugins directory:

Windows / macOS:
<IDA install dir>/plugins/

Linux:
~/.idapro/plugins/

Restart IDA Pro.

FEATURES

Export function signatures to .cfs files

Import signatures and automatically rename functions

Supports wildcard matching (??)

Safe renaming (auto-named functions only by default)

Configurable import and export options

.CFS FILE FORMAT

The .cfs file uses a CSV-like format:

index,"function_name","AA BB CC ?? DD"

Field description:

index
Function order index

function_name
Name that will be applied in IDA

signature_hex
Hex byte signature (?? is a wildcard)

Example:

0,"InitNetwork","55 8B EC 83 EC ?? 53 56"
1,"HandlePacket","48 89 ?? ?? ?? 57 41 56"

USAGE

EXPORT SIGNATURES

Open an IDB in IDA.

Select one or more functions, or leave empty to export all.

Use one of the following:

File → Produce file → CFS Export…

Right-click in the Functions window → CFS Export…

Shortcut: Ctrl + Shift + E

Configure export options and save the .cfs file.

IMPORT SIGNATURES

Open the target IDB in IDA.

Use one of the following:

File → Load file → CFS Import…

Right-click in the Functions window → CFS Import…

Shortcut: Ctrl + Shift + I

Select the .cfs file.

Matching functions will be renamed and tagged.

SETTINGS

Open:
Edit → Plugins → CFS Settings…

Available options:

Export max bytes

Export only auto-named functions (sub_, nullsub_, j_)

Search executable segments first

Mask immediates and references

Minimum signature length

BEST PRACTICES

Export signatures from older builds and import into newer versions.

Use masking to minimize differences between builds.

Do not import random .cfs files unless you trust them.

Reduce export size (e.g. 32 bytes) if signatures become unstable.

SHORTCUTS

Import: Ctrl + Shift + I
Export: Ctrl + Shift + E
Settings: Edit → Plugins → CFS Settings…
